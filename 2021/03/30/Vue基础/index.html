Vue基础

发表于 2021-03-30 更新于 2021-03-31 分类于 study ， Vue

1. 声明

  1. 创建一个Vue实例：

    var app2 = new Vue({
	el: '#app',
    data: {
        message: ''
    }
})

  1. 实例的生命周期钩子：

    1. created：在实例被创建之后执行

    2. mounted：在实例被挂载的时候执行

    3. updated：在实例被更新时执行

    4. destroyed：在实例被销毁后执行

    5. 不要在选项 property 或回调上使用箭头函数，比如

      created: () => console.log(this.a) 或 vm.$watch('a', newValue => this.myMethod())。/*因为箭头函数并没有 this，this 会作为变量一直向上级词法作用域查找，直至找到为止，经常导致 Uncaught TypeError: Cannot read property of undefined 或 Uncaught TypeError: this.myMethod is not a function 之类的错误。*/

2. 模板语法

  1. 使用：{{}} ,双大括号形式使用，Mustache语法

  2. 指令

    • v-once：一次性的插入
    • v-html：将数据解析成html代码
    • v-if：条件判断
      • v-else
      • v-if-else
    • v-on(@)：绑定事件
    • v-bind(:)：响应式更新

  3. 动态参数：

    <a v-bind:[attributeName]="url"> ... </a>

3. 计算属性

  1. 用于存放逻辑运算的属性

    <div id="example">
  <p>Original message: "{{ message }}"</p>
  <p>Computed reversed message: "{{ reversedMessage }}"</p>
</div>

<script>
    var vm = new Vue({
      el: '#example',
      data: {
        message: 'Hello'
      },
      computed: {
        // 计算属性的 getter
        reversedMessage: function () {
          // `this` 指向 vm 实例
          return this.message.split('').reverse().join('')
        }
      }
    })
</script>

  2. vs方法

    1. 计算属性基于响应式依赖进行缓存。可用于多次重复访问。
    2. 方法可以不依赖响应式，每次调用都会重新执行。

  3. 计算属性 VS 侦听属性

    1. 侦听属性：watch

      //在vue实例中...
watch: {
	firstName: function (val) {
      this.fullName = val + ' ' + this.lastName
    },
    lastName: function (val) {
      this.fullName = this.firstName + ' ' + val
    }
}

  4. 计算属性setter

    1. 被设置时会更新

4. 侦听器

  1. 虽然计算属性在大多数情况下更合适，但有时也需要一个自定义的侦听器。

  2. 作用：响应数据的变化

  3. 当数据变化时执行异步或开销较大的操作时，最有用。

  4. 列如：

    <div id="watch-example">
  <p>
    Ask a yes/no question:
    <input v-model="question">
  </p>
  <p>{{ answer }}</p>
</div>
<script>
	// vue实例中...
    watch: {
    // 如果 `question` 发生改变，这个函数就会运行
    question: function (newQuestion, oldQuestion) {
      this.answer = 'Waiting for you to stop typing...'
      this.debouncedGetAnswer()
    }
  },
</script>

  5. 我们得到最终结果前，设置中间状态。这些都是计算属性无法做到的。

5. Class 与 Style 绑定

  1. 动态切换class：

    <div v-bind:class="{active: isActive}">
 
</div>

  2. 多个字段

    <div v-bind:class="{ active: isActive}, 'text-danger':hasError}">
    
</div>
<scrit>
    // vue实例中
	data: {
    	isActive: true,
    	hasError: true
    },
    computed: {
    	classObject: function() {
    		return {
    			active: this.isActive && !this.error,
    			'text-danger': this.error && this.error.type === 'fatal'
        	}
    	}
    }
</scrit>
// 结果渲染为
<div class="active text-danger">
    
</div>

7. 列表渲染

  1. 基本使用

    <li v-for="item in items" :key="item.message">
    {{ item.message }}
  </li>

  2. 可选第二个参数，索引

    <li v-for="(item, index) in items">
  {{ parentMessage }} - {{ index }} - {{ item.message }}
</li>

  3. of 替代 in 作为分隔符

    <div v-for="item of items"></div>

  4. 遍历对象

    // 值，属性
<div v-for="(value, name) in object">
  {{ name }}: {{ value }}
</div>
// 值，属性，下标
<div v-for="(value, name, index) in object">
  {{ index }}. {{ name }}: {{ value }}
</div>

8. 事件处理

  1. 监听事件：v-on

    <button v-on:click="counter += 1">Add 1</button>

  2. 事件处理方法

    • event: 原生DOM事件
    • this：指向Vue实例

  3. 内联处理器中的方法

    1. 直接绑定<button v-on:click="say('hi')">Say hi</button>

    2. $event：DOM事件

      <button v-on:click="warn('Form cannot be submitted yet.', $event)">
  Submit
</button>

  4. 事件修饰符

    • .stop

    • .prevent

    • .capture

    • .self

    • .once

    • .passive

      <!-- 阻止单击事件继续传播 -->
<a v-on:click.stop="doThis"></a>
<!-- 提交事件不再重载页面 -->
<form v-on:submit.prevent="onSubmit"></form>

<!-- 修饰符可以串联 -->
<a v-on:click.stop.prevent="doThat"></a>

<!-- 只有修饰符 -->
<form v-on:submit.prevent></form>

<!-- 添加事件监听器时使用事件捕获模式 -->
<!-- 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理 -->
<div v-on:click.capture="doThis">...</div>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThat">...</div>

    • v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。

  5. Vue 还对应 addEventListener 中的 passive 选项提供了 .passive 修饰符。

    <!-- 滚动事件的默认行为 (即滚动行为) 将会立即触发 -->
<!-- 而不会等待 `onScroll` 完成  -->
<!-- 这其中包含 `event.preventDefault()` 的情况 -->
<div v-on:scroll.passive="onScroll">...</div>

  6. 按键修饰符

    1. keyCode

      • .enter
      • .tab
      • .delete (捕获“删除”和“退格”键)
      • .esc
      • .space
      • .up
      • .down
      • .left
      • .right

    2. config.keyCodes 对象自定义按键修饰符别名：

      Vue.config.keyCodes.f1 = 112

    3. 系统修饰符

      • .ctrl

      • .alt

      • .shift

      • .meta

        <!-- Alt + C -->
<input v-on:keyup.alt.67="clear">

<!-- Ctrl + Click -->
<div v-on:click.ctrl="doSomething">Do something</div>

    4. exact修饰符

      1. .exact 修饰符允许你控制由精确的系统修饰符组合触发的事件。

        <!-- 即使 Alt 或 Shift 被一同按下时也会触发 -->
<button v-on:click.ctrl="onClick">A</button>

<!-- 有且只有 Ctrl 被按下的时候才触发 -->
<button v-on:click.ctrl.exact="onCtrlClick">A</button>

<!-- 没有任何系统修饰符被按下的时候才触发 -->
<button v-on:click.exact="onClick">A</button>

    5. 鼠标按钮修饰符

      • .left
      • .right
      • .middle
